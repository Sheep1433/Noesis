# 单文件上传需求（虚构）

## 1. 背景

知识库与工单模块需要统一的 **文件上传** 能力，供用户提交需求文档、截图与附件。本期为单文件同步上传接口，大文件分片见 UPLOAD-CHUNK 文档，病毒扫描见 UPLOAD-AV 文档。本文不涉及登录、审批流配置。

## 2. 支持格式与大小

| 类型 | 扩展名 | 单文件上限 |
|------|--------|------------|
| 文档 | PDF、DOCX、Markdown（.md） | **20MB** |
| 图片 | PNG、JPG | 10MB（本期可合并到同一接口校验） |

- 上传前前端须校验扩展名与大小；**超过 20MB 拦截** 且不发起请求，提示「文件不能超过 20MB」
- 服务端二次校验 MIME 与魔数，防止伪造扩展名

## 3. 上传流程

1. 用户在选择文件后看到本地文件名、大小预览
2. 点击「上传」，显示进度条（0–100%）
3. 成功：返回 `file_id`、`file_name`、`size`、`checksum`（SHA256）
4. 失败：返回可读错误码与中文说明

## 4. 异常处理

| 场景 | 处理 |
|------|------|
| 不支持格式 | **不支持格式提示**：列出允许的类型，如「仅支持 PDF、DOCX、Markdown」 |
| 超过大小限制 | 前后端一致拦截，错误码 `FILE_TOO_LARGE` |
| 网络中断 | 允许重试；已写入部分数据时服务端清理孤儿文件 |
| 存储服务不可用 | 503，提示「上传服务暂不可用，请稍后重试」 |

## 5. 存储与权限

- 对象存储路径：`/{tenant_id}/uploads/{yyyy}/{mm}/{file_id}`
- 下载须带签名 URL，有效期 15 分钟
- 仅上传者及同租户管理员可删除

## 6. 审计

记录：操作人、原始文件名、大小、上传时间、关联业务 ID（可选）

## 7. 性能指标

- 20MB 文件在 100Mbps 内网环境 P95 完成时间 &lt; 30 秒
- 接口支持 Idempotency-Key 防止重复提交
