# 企业微信 OAuth 登录需求（虚构）

## 1. 概述

为满足企业客户「扫码即用」习惯，系统在保留账号密码登录的前提下，新增 **企业微信扫码登录**。用户使用手机企业微信扫描二维码后，由本系统完成 OAuth2 授权码换票、本地用户绑定与登录态签发。本需求不涉及支付、收银台或订单创建能力。

## 2. 角色与前置条件

- **企业管理员**：在管理后台完成 CorpID、AgentID、Secret 配置并启用「企业微信登录」
- **终端用户**：已安装企业微信且加入对应企业通讯录；若首次使用须完成 **绑定本地账号**

## 3. 主流程：扫码登录

1. 登录页展示「企业微信扫码登录」二维码（有效期 2 分钟，超时自动刷新）
2. 用户使用企业微信扫一扫，确认授权
3. 回调携带 `code`，后端向企业微信接口换取 `userid`、姓名、部门等
4. 若 `userid` 已绑定本地 `user_id`：直接登录，跳转首页
5. 若未绑定：进入「绑定本地账号」页，要求输入工号+密码或邀请码完成 **首次绑定本地账号**，绑定成功后自动登录

## 4. 异常流程

| 编号 | 场景 | 期望行为 |
|------|------|------|
| E1 | 用户取消扫码 | 返回登录页，Toast「已取消扫码」 |
| E2 | 二维码过期 | 提示「二维码已过期，请刷新」并重新生成 |
| E3 | 绑定失败（工号不存在/密码错误） | 明确提示原因，不泄露其他用户信息 |
| E4 | 企业未开通该应用 | 提示「企业未授权，请联系管理员」 |

## 5. 安全与审计

- `code` 一次性使用，5 分钟内有效
- 绑定操作须二次验证本地密码
- 审计日志记录：扫码时间、userid、绑定结果、IP

## 6. 接口与兼容

- 回调 URL 固定为 `https://{domain}/api/auth/wework/callback`
- 须兼容 Chrome 90+、Edge 90+；移动端 H5 同流程

## 7. 验收标准摘要

- 已绑定用户扫码后 3 步内进入首页
- 未绑定用户必须完成绑定才能访问业务接口
- 全流程文案为中文，关键按钮具备加载态防止重复提交
