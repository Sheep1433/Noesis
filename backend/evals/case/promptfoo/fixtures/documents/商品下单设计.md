# 商品下单功能设计文档

## 1. 概述

### 1.1 功能简介

商品下单模块负责管理用户的购物车和订单创建，支持添加商品、修改数量、确认订单、支付等完整购物流程。

### 1.2 核心特性

| 特性 | 说明 |
|------|------|
| 购物车管理 | 添加/删除/修改商品数量 |
| 订单创建 | 从购物车确认生成正式订单 |
| 库存扣减 | 下单时实时扣减库存 |
| 订单状态 | 待支付/已支付/已发货/已完成/已取消 |
| 价格计算 | 支持商品单价、数量、折扣计算 |

## 4. API 设计

### 4.1 购物车相关

#### GET /api/cart
获取当前用户购物车。

**响应：**
```json
{
  "code": 200,
  "msg": "获取购物车成功",
  "data": {
    "id": "cart-uuid",
    "items": [
      {
        "id": "item-uuid",
        "product_id": "product-uuid",
        "product_name": "商品名称",
        "quantity": 2,
        "price": 99.00
      }
    ],
    "total_amount": 198.00
  }
}
```

#### POST /api/cart/items
添加商品到购物车。

**请求体：**
```json
{
  "product_id": "product-uuid",
  "quantity": 1
}
```

#### PUT /api/cart/items/{item_id}
修改购物车商品数量。

**请求体：**
```json
{
  "quantity": 3
}
```

#### DELETE /api/cart/items/{item_id}
删除购物车商品。

#### POST /api/orders
创建订单（从购物车确认）。

**请求体：**

**响应：**

获取当前用户订单列表。

#### GET /api/orders/{order_id}

#### PUT /api/orders/{order_id}/pay
订单支付。

**请求体：**
```json
{
  "pay_method": "alipay"
}
```

#### PUT /api/orders/{order_id}/cancel
取消订单。

## 5. 核心功能流程

### 5.1 添加购物车流程

```
1. 用户点击"加入购物车"
   │
2. 前端调用 POST /api/cart/items
   │
3. 后端检查商品库存是否充足
   │
4. 检查购物车是否已有该商品
   │   ├── 有 → 更新数量 +1
   │   └── 无 → 新增购物车商品记录
   │
5. 返回更新后的购物车
```

**关键代码：** `CartService.add_item()`
```python
# 检查库存
product = await db.execute(select(TProduct).where(TProduct.id == product_id))
if product.stock < quantity:
    raise ServiceException(message='库存不足')

# 检查是否已存在
existing = await db.execute(
    select(TCartItem).where(
        TCartItem.cart_id == cart_id,
        TCartItem.product_id == product_id
    )
)
if existing:
    existing.quantity += quantity
else:
    item = TCartItem(
        id=str(uuid.uuid4()),
        cart_id=cart_id,
        product_id=product_id,
        quantity=quantity,
        price=product.price
    )
    db.add(item)
```

### 5.2 创建订单流程

```
1. 用户确认购物车商品，点击"提交订单"
   │
2. 前端调用 POST /api/orders
   │
3. 后端开启事务
   │
4. 锁定购物车商品，生成订单号
   │
5. 遍历购物车商品：
   │   ├── 检查库存是否充足
   │   ├── 扣减库存
   │   └── 创建订单商品记录
   │
6. 计算订单总金额
   │
7. 创建订单记录
   │
8. 清空已结算的购物车商品
   │
9. 提交事务，返回订单信息
```

**关键代码：** `OrderService.create_from_cart()`
```python
async with db.begin():
    # 生成订单号
    order_no = f"ORD{datetime.now().strftime('%Y%m%d%H%M%S')}"

    # 创建订单
    order = TOrder(
        id=str(uuid.uuid4()),
        order_no=order_no,
        user_id=user_id,
        total_amount=0,
        status='pending'
    )
    db.add(order)

    # 处理购物车商品
    total = 0
    for item in cart_items:
        # 扣减库存
        await db.execute(
            update(TProduct)
            .where(TProduct.id == item.product_id)
            .values(stock=TProduct.stock - item.quantity)
        )

        # 创建订单商品
        order_item = TOrderItem(
            id=str(uuid.uuid4()),
            order_id=order.id,
            product_id=item.product_id,
            product_name=item.product_name,
            quantity=item.quantity,
            price=item.price,
            subtotal=item.quantity * item.price
        )
        db.add(order_item)
        total += order_item.subtotal

    order.total_amount = total
```

### 5.3 订单支付流程

```
1. 用户选择支付方式，点击"支付"
   │
2. 前端调用 PUT /api/orders/{order_id}/pay
   │
3. 后端检查订单状态是否为 pending
   │
4. 调用支付网关（模拟）
   │
5. 支付成功后：
   │   ├── 更新订单状态为 paid
   │   ├── 记录支付时间
   │   └── 返回支付结果
   │
6. 前端跳转至支付成功页
```
