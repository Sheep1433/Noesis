# 会话保持与登出需求（虚构）

## 1. 文档目的

定义用户登录后的 **会话（Session）** 生命周期：「记住我」长期登录、浏览器关闭行为、主动登出及多标签页一致性。本文档不描述文件上传、审批等业务模块。

## 2. 术语

- **会话令牌**：HttpOnly Cookie 中的 `session_token`，或由前端持有的等价凭证（实现可选 JWT + Refresh）
- **记住我**：用户勾选后延长会话有效期；未勾选则为浏览器会话级

## 3. 记住我

| 勾选状态 | 会话有效期 | 说明 |
|----------|------------|------|
| 未勾选 | 浏览器关闭即失效 | 不写入持久化 Cookie |
| 已勾选 | **7 天** | 滑动续期：活跃用户每日访问自动续 7 天 |

- 勾选「记住我」须在登录接口传 `remember_me=true`
- 7 天到期后跳转登录页，提示「登录已过期，请重新登录」
- 用户可在「个人设置」中查看最近登录设备并撤销指定设备会话

## 4. 主动登出

1. 用户在设置页点击「退出登录」
2. 服务端将当前 `session_token` 加入黑名单，清除服务端会话
3. 前端清除本地缓存（含 Pinia/LocalStorage 中的用户概要，如有）
4. **登出后其它标签页失效**：其它已打开标签在下次 API 请求收到 401 时，统一跳转 `/login` 并提示「您已在其他窗口退出」

## 5. 多标签与并发

- 同一账号允许多端同时在线（默认最多 5 端，可配置）
- 登出仅失效当前端会话时，须在需求澄清会确认；**本版本要求：单端登出失效当前 token，其它标签共享同一 token 则一并失效**

## 6. 异常与边界

- 会话被管理员强制下线：返回 401，文案「账号已被管理员下线」
- 时钟 skew：服务端以 UTC 校验过期时间，容忍客户端 ±120 秒

## 7. 非功能

- 登出接口须幂等，重复调用仍返回 200
- 会话状态变更须在 1 秒内对网关可见（避免登出后仍能调用旧接口）
