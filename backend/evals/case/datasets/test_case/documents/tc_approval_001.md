# 员工请假审批流程需求（虚构）

## 1. 业务背景

HR 系统需支持员工在线提交请假单，经直属主管 **审批** 后生效，并同步考勤系统。本期覆盖事假、病假、年假三种类型，不含薪资计算与 **文件上传** 附件病毒扫描（附件走通用上传模块）。

## 2. 流程概览

```
员工起草 → 提交 → 直属主管审批 → （年假）HR 备案 → 结束
                ↘ 驳回 → 员工修改重提
```

## 3. 请假类型与规则

| 类型 | 规则 |
|------|------|
| 事假 | 提前 0.5 个工作日提交；单次不超过 5 天 |
| 病假 | 超过 3 天须上传医院证明（附件可选） |
| 年假 | **年假提前 1 天提交**；余额不足不允许提交 |

- 审批人：员工档案中的 `direct_manager_id`，若为空则转 HR 值班组
- 并发规则：同一自然日不允许重叠的有效请假单

## 4. 审批操作

- **通过**：填写可选意见，状态 `approved`，写入考勤
- **驳回**：**驳回须填原因**，原因必填且 5–200 字；状态 `rejected`，通知员工

员工收到驳回后可修改除「工号、姓名」外的字段重新提交，生成新版本，保留历史。

## 5. 状态机

`draft` → `pending` → `approved` | `rejected`  
`rejected` → 修改后 → `pending`

- `pending` 超过 72 小时未处理，系统提醒审批人；不自动通过

## 6. 通知

- 提交、通过、驳回均发送邮件+站内信
- 抄送 HR（仅年假备案节点）

## 7. 权限

- 员工仅查看本人单据
- 主管查看下属待办与已办
- HR 可查看全租户统计，不可篡改已审批记录

## 8. 非功能

- 提交接口幂等
- 审批操作须记录操作人、时间、IP

## 9. 界面要点（草案）

- 待办列表按提交时间倒序，支持按类型筛选
- 详情页展示审批轨迹时间轴
- 移动端仅支持审批与查看，不支持批量导入年假余额

## 10. 不在范围

采购申请、报销单、加班调休（另立项）。
