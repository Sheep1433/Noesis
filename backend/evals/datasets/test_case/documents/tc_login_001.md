# 用户登录模块需求规格说明（虚构）

| 版本 | 日期 | 作者 |
|------|------|------|
| V1.2 | 2026-03-01 | 产品组 |

## 1. 背景与目标

统一身份认证网关面向 ToB 企业客户，需支持账号密码登录，并通过图形验证码降低撞库与暴力破解风险。本模块不包含注册、找回密码（另见 UAM-REG 文档），也不涉及支付、订单等业务流程。

## 2. 功能范围

### 2.1 登录入口

- Web 端登录页路径：`/login`
- 必填项：用户名（支持手机号或工号）、密码、图形验证码
- 可选：「记住我」勾选框（会话策略见 UAM-SESSION 文档，本模块仅透传参数）

### 2.2 正常登录流程

1. 用户打开登录页，系统生成 4 位字母数字混合验证码图片，同时下发 `captcha_id`（有效期 5 分钟）
2. 用户填写用户名、密码、验证码后点击「登录」
3. 服务端校验：验证码未过期 → 用户名密码匹配 → 签发 JWT，跳转首页 `/dashboard`
4. 登录成功写入审计日志：时间、IP、设备指纹（脱敏展示）

### 2.3 异常与提示文案（须严格一致）

| 场景 | 提示 |
|------|------|
| 用户名或密码任一错误 | 「用户名或密码错误」（不得提示「用户名不存在」或「密码错误」） |
| 验证码输入错误 | 「验证码错误，请重新输入」 |
| 验证码过期 | 「验证码已过期，请刷新」；前端须支持一键刷新验证码 |
| 账号锁定 | 「账号已锁定，请 15 分钟后重试或联系管理员」 |

### 2.4 安全策略

- 同一用户名连续登录失败 **5 次**（含密码错误、验证码错误均计入），锁定 **15 分钟**
- 锁定期间即使密码正确亦拒绝登录
- 密码传输须 HTTPS；后端仅存加盐哈希，日志禁止打印明文密码

## 3. 非功能需求

- 登录接口 P99 响应 &lt; 800ms（不含短信）
- 验证码图片须可被屏幕阅读器识别为「验证码图片」并支持刷新按钮无障碍标签

## 4. 不在范围

支付回调、订单查询、第三方 OAuth（见 UAM-OAUTH）。
