# 上传文件病毒扫描与隔离需求（虚构）

## 1. 目标

所有经上传通道进入对象存储的文件，在可被业务引用或 **下载** 之前，必须经过异步 **病毒扫描** 与策略引擎判定，防止恶意样本进入知识库与工单系统。本模块与用户 **登录** 认证解耦，假定上传请求已通过网关鉴权。

## 2. 扫描流程

1. 文件直传或分片合并完成后，写入「待扫描」队列，业务侧 `file_status=pending_scan`
2. 扫描引擎（对接虚构 AV-Agent）拉取对象，返回：clean / infected / error
3. clean：状态改为 `available`，允许预览与下载
4. infected：进入 **隔离区**，`file_status=quarantined`
5. error：最多重试 3 次，仍失败则 `scan_failed`，人工复核

## 3. 隔离策略

- **高危文件隔离**：判定为木马、勒索、宏病毒等，立即隔离，不可被任何业务 API 引用
- 隔离对象移至独立 bucket 前缀 `/quarantine/`，原路径失效
- 用户侧列表展示状态「审核中」或「存在风险已隔离」，不暴露病毒名细节
- **隔离文件不可下载**：对 `quarantined` 状态返回 403，文案「文件存在安全风险，无法下载」

## 4. 通知与申诉

- 上传者收到站内信：文件名、时间、处理结果
- 企业管理员可在后台查看隔离列表，支持误报申诉（二期人工放行，本期仅记录工单号）

## 5. 性能与 SLA

- 50MB 以内文件，扫描完成 P95 &lt; 3 分钟
- 队列堆积超过 1000 须告警

## 6. 合规

- 扫描日志保留 180 天：file_id、hash、结果、引擎版本
- 不得将文件内容外传至境外节点（部署约束）

## 7. 接口约束

- `GET /files/{id}/download` 须校验 `file_status==available`
- 预览接口与下载接口规则一致

## 8. 与上传模块衔接

- 直传与分片合并完成事件均须投递同一扫描队列
- 扫描中的文件在业务侧展示「处理中」，禁止关联到知识库索引任务
