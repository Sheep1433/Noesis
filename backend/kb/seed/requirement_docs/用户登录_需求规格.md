# 用户登录功能需求规格说明书

> 文档编号：REQ-AUTH-2024-003 | 版本：v1.0 | 状态：已归档

## 1. 概述

用户登录模块提供账号密码认证能力，登录成功后签发 JWT token，供后续购物车、订单等接口鉴权使用。

## 2. 登录接口

### 2.1 基本信息

- **路径**：`POST /api/user/login`
- **Content-Type**：`application/x-www-form-urlencoded`（OAuth2 Password Flow）
- **响应**：`{"token": "...", "token_type": "bearer"}`

### 2.2 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 用户名，3-50 字符 |
| password | string | 是 | 密码，8-32 字符 |

## 3. 业务规则

1. 用户名不存在时返回「用户不存在」，HTTP 401
2. 密码错误时返回「密码错误」，HTTP 401
3. 连续 5 次密码错误锁定账号 15 分钟
4. 登录成功更新 `last_login_at` 字段
5. token 有效期 24 小时，过期需重新登录
6. 密码存储使用 bcrypt 哈希，禁止明文

## 4. 鉴权规范

- 业务接口通过 `Authorization: Bearer {token}` 传递
- token 无效或过期返回 HTTP 401
- 未登录访问需鉴权接口返回「未授权」

## 5. 安全要求

- 登录接口限流：同一 IP 每分钟最多 20 次
- 密码错误提示不得泄露用户名是否存在（对外统一文案，内网日志可区分）
- HTTPS 传输
