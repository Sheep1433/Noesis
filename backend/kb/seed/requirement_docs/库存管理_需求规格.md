# 库存管理功能需求规格说明书

> 文档编号：REQ-STOCK-2024-002 | 版本：v1.0 | 状态：已归档

## 1. 概述

库存管理模块维护商品可售数量，在加购、改量、下单、取消订单等场景进行库存校验与扣减/回滚。

## 2. 数据模型

### 2.1 库存表 t_stock

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(36) | 主键 |
| product_id | VARCHAR(36) | 关联商品 |
| quantity | INT | 可售库存，≥ 0 |
| locked_quantity | INT | 预扣库存（下单未支付） |
| updated_at | BIGINT | 更新时间戳 |

## 3. 库存扣减规则

### 3.1 加购校验

- 加购数量 ≤ 当前可售库存（`quantity - locked_quantity`）
- 库存为 0 时拒绝加购，提示「库存不足」

### 3.2 下单扣减

1. 创建订单时：`quantity -= order_qty`，`locked_quantity += order_qty`（可选预扣策略）
2. v1.0 简化策略：下单直接扣减 `quantity`，不维护 `locked_quantity`
3. 扣减使用乐观锁：`UPDATE ... WHERE quantity >= ? AND version = ?`
4. 扣减失败整单回滚

### 3.3 取消回滚

- 待支付订单取消：`quantity += order_qty`
- 支付超时自动取消同样回滚库存

## 4. 边界场景

| 场景 | 预期行为 |
|------|----------|
| 库存刚好等于购买量 | 允许下单，库存变为 0 |
| 并发两人抢最后 1 件 | 仅一人成功，另一人「库存不足」 |
| 商品已下架 | 拒绝加购与下单 |
| 数量为 0 或负数 | 参数校验失败 |

## 5. 接口依赖

- 加购：`POST /api/cart/items` 调用库存校验
- 改量：`PUT /api/cart/items/{id}` 校验新数量
- 下单：`POST /api/orders` 批量扣减
