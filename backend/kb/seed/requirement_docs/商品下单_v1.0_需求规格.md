# 商品下单功能需求规格说明书 v1.0

> 文档编号：REQ-ORDER-2024-001 | 版本：v1.0 | 状态：已归档

## 1. 概述

商品下单模块负责用户从购物车确认商品、生成订单并完成支付的完整流程。本版本为 v1.0 基线需求，后续 v1.1 补充了库存预扣与订单超时取消规则。

### 1.1 业务目标

- 支持用户将购物车商品一键生成订单
- 下单时实时校验库存并扣减
- 订单金额按商品单价 × 数量累加，保留 2 位小数
- 支持待支付、已支付、已取消三种核心状态

### 1.2 范围

| 模块 | 是否包含 |
|------|----------|
| 购物车管理 | 否（见 REQ-CART-2024-001） |
| 订单创建 | 是 |
| 订单支付 | 是 |
| 库存扣减 | 是 |
| 物流发货 | 否（v2.0 规划） |

## 2. 订单创建

### 2.1 接口

- **路径**：`POST /api/orders`
- **鉴权**：需要登录 token
- **请求体**：`{"cart_id": "cart-uuid-001"}`

### 2.2 业务规则

1. 仅 `status=active` 的购物车可生成订单
2. 购物车不能为空，否则返回 400「购物车为空」
3. 每个商品行数量必须 > 0
4. 下单前校验每个 SKU 库存 ≥ 购买数量，不足返回「库存不足」
5. 订单 `total_amount` = Σ(单价 × 数量)，金额精度 2 位小数
6. 下单成功后购物车状态变为 `checked_out`
7. 库存立即扣减，扣减失败则整单回滚

### 2.3 订单金额计算

```
total_amount = sum(item.unit_price * item.quantity for item in cart_items)
```

- 单价取自下单时刻商品表 `t_product.price`
- 不允许在订单层再次应用折扣（v1.0 不支持优惠券）
- 金额比较使用 `Decimal`，禁止浮点累加误差

## 3. 订单支付

### 3.1 接口

- **路径**：`POST /api/orders/{order_id}/pay`
- **请求体**：`{"payment_method": "alipay" | "wechat" | "balance"}`

### 3.2 业务规则

1. 仅 `status=pending_payment` 的订单可支付
2. 支付成功后状态变为 `paid`，记录 `paid_at` 时间戳
3. 支付超时（30 分钟）未支付自动取消并恢复库存
4. 重复支付请求需幂等，返回已有支付结果

## 4. 订单取消

### 4.1 规则

- 待支付订单：用户可主动取消，恢复库存
- 已支付订单：v1.0 不支持用户自助取消，需走售后流程
- 取消后订单状态为 `cancelled`

## 5. 非功能需求

- 下单接口 P99 响应时间 < 500ms
- 同一用户并发下单需防重复提交（幂等键 `Idempotency-Key`）
- 所有金额字段数据库使用 `DECIMAL(12,2)`
